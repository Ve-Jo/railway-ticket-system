# Демо-данные

Файл [db/seeds.sql](C:/Users/PAPA/Downloads/diploma/db/seeds.sql) добавляет минимальный набор данных для локальной демонстрации `MVP`:

- роли пользователей `passenger`, `cashier`, `admin`;
- 3 тестовых аккаунта;
- 5 станций;
- 2 маршрута;
- 1 поезд;
- 2 вагона;
- 16 мест;
- 3 рейса.

## Тестовые аккаунты

Для всех трёх аккаунтов используется один и тот же пароль: `Demo123!`

- `passenger_demo` / `passenger.demo@example.com`
- `cashier_demo` / `cashier.demo@example.com`
- `admin_demo` / `admin.demo@example.com`

Пароли в `db/seeds.sql` уже сохранены в виде `bcrypt`-хэша. Это совместимо с текущим backend, где проверка выполняется через `bcryptjs`.

## Порядок загрузки

1. Создать пустую базу MySQL, например `railway_ticket_system`.
2. Применить схему:

```powershell
mysql -u root -p railway_ticket_system < db/schema.sql
```

3. Загрузить демо-данные:

```powershell
mysql -u root -p railway_ticket_system < db/seeds.sql
```

## Что можно показать на демо

- вход под пассажиром, кассиром и админом;
- поиск рейсов между Киевом и Львовом;
- отображение вагонов и мест для одного поезда;
- работу ролей и будущие сценарии бронирования, оплаты и возврата.

## Ограничения

- В сиды не добавлены `bookings`, `tickets`, `payments`, `refunds` и `operation_logs`: эти данные лучше создавать уже через приложение, чтобы демонстрировать реальный жизненный цикл билета.
- Даты рейсов в сид-файле фиксированные: `2026-07-01`, `2026-07-02`, `2026-07-03`. Если демонстрация будет сильно позже, даты в `db/seeds.sql` стоит обновить.
- Сид-файл рассчитан на повторный запуск без дублей по основным уникальным ключам, но не предназначен для очистки существующих бизнес-данных.
